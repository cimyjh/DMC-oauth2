포트폴리오 스펙

  • Framework: Spring boot 2.3
  • WAS: AWS(Amazon WebService)의 EC2
  • DB Servcer: AWS(Amazon WebService)의 RDS
  • ORM(Object-relational mapping): JPA_Java Persistence API
  • JPA 보완 기술: Querydsl
  • 회원관리: Spring Security_OAuth2인증(Google, Naver)
  • Data Insert: python의 BeautifulSoup을 활용한 웹크롤링
  • Server side: thymeleaf
  • API: RESTful API

        저는 창의적인 생각과 발상의 전환에 대한 아이디어를 실제로 구현해본 경험이 있습니다.
      학부 4학년 전공 수업에서 텍스트마이닝과 통계적 기법을 활용한 R 프로그래밍을 교육받았습니다. 데이터를 수집하여 제가 원하는 정보를 직접 구현해볼 수 있다는 점에 매료되었습니다. 해당 수업 이수 후에 저는 수업의 연장선으로 개인 프로젝트를 진행했습니다.
      개인 프로젝트의 주제는 증권 투자기법 기술적 분석에 대한 검증과 효과적인 독자적인 합성전략을 찾는 것이었습니다. 해당 프로젝트는 400시간 이상 작업했으며, 시행착오 끝에 결과물을 도출하고 종료했습니다. 종료 이후에 지금도 취미 활동으로 토이 프로그래밍으로 가끔 작업하고 있습니다.
     해당 프로젝트는 코스피, 코스닥 각각 상위 30종목을 대상으로 했습니다. 총 18가지 기술적 지표와 63개의 시그널을 검증했습니다. 검증 방식은 제가 독자적으로 설계했습니다. 매수, 매도 시그널 5일 후의 5일 이동평균을 시그널 당일 종가로 나누는 형식으로 설계했습니다. 위 검증방식은 시그널에 대한 단기지표를 검증하는 데 효과적이었습니다. 데이터는 키움증권 OpenAPI를 파이썬으로 접근하여 수집했습니다. 수집된 OHLCV 데이터를 R을 통해 통계작업을 했었습니다.
     검증 방식은 어느 HTS에서도 제공하지 않는 창의적인 기술이었으며, 프로젝트의 결과로서 거래량 지표인 VR과 가격지표의 Stochastic O의 합성이 가장 효과적인 것을 도출했을 때, 저는 핀테크 분야에 대한 흥미와 프로그래밍에 대한 재미를 느꼈습니다. 또한 제가 좋아하는 일이 제일 잘 할 수 있는 일이라는 것을 깨달았습니다.
